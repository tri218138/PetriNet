Symbol Name Application
Place

A place ùëù is represented by a circle. A place ùëù always models a passive component.

ùëù can store, accumulate or show things. A place is a discrete state.
Transition

A transition ùë° is represented by a square or rectangle. A transition ùë° always models an active component.

ùë° can produce things/tokens, consume, transport or change them through the Firing mechanism

Arrow

Places and transitions are connected to each other by directed arcs, graphically, represented by an arrow.

An arc never models a system component, sometimes only notional relation between components such as logical connections.
Token

A token is a special transition node, being graphically rendered as a black dot.

Symbolic tokens generally denote elements of the real world. Places can contain tokens, and transitions where hold and wait for firing tokens
Example Model

Network is static and composed of place and transitions.

Places hold tokens.

Functions of Model

Button Application
Run

Run the pre-built program built on the Petri-Net model of specialists and patients

When the program runs successfully, it will say "Process done! ".When deadlock-free the program will stop and display the message "deadlock-free"
Add Token

Select a place and then press the button to add the token to that place.

With the number of places less than 10, the tokens will show a black dot. Opposite,number of places equal or greater than 10, tokens will be displayed as quantity
Remove Token

Select a place and then press the button to delete the token in that place.

Can not delete token if there is not any tokens
Speed

Increase the speed of the "animation arrow" effect
